# curl

* **curl（全称 Client URL）** 是一个跨平台的网络请求工具 / 编程库，核心作用就是：**不用浏览器，也能按照各种网络协议发送请求、获取数据**。

* 最简单的 curl 命令格式：

```bash
curl 网址
```

例子：访问百度

```bash
curl https://www.baidu.com
```

* --path-as-is = 禁止 curl 乱动我的网址路径！你让我发什么，就原封不动发给服务器！
  * 不加这个参数，curl 会「多管闲事」
    curl 默认是个勤快过头的小管家，它看到网址里的路径，会自动帮你整理、简化、修改。
  * 你想发路径：/config/app/dev/../flag，curl 一看：哦？../ 是返回上一级，我帮你简化一下！→ 自动改成：/config/app/flag→ 发给服务器的是简化后的干净路径
  * 加密乱码 ..%252F，curl 会自动解析它、修改它，把你的攻击路径直接改没了！
* ..%252F = 编码后的../，用来返回上一级文件夹，偷读根目录的 flag
  * 网址里不能直接写 /，会被编码成 %2F
  * 网址里不能直接写 %，会被编码成 %25
  * 所以：%252F = 双重编码的 /
  * 服务器会拦截 / 和它的单次编码，双重编码是为了骗过服务器的防火墙 / 过滤规则 ，让它放行，最后自己变回 /
