### 伪协议

什么叫「伪协议」
先分两种协议：

1. 真协议（你天天在用）
    http:// https:// ftp://
    • 作用：上网、访问外网地址
    • 是互联网通用的正规协议
2. PHP 伪协议 php://
    • 不是用来上网的
    • 是 PHP 自己内部私有的一条通道
    • 长得像协议（带 ://），但不是网络协议，是 PHP 内部专用
    👉 长得像协议、但不是正经网络协议，所以叫 伪协议
    你就记：php:// 开头的，都是 PHP 偷偷自带的内部小路子，专门用来读文件、拿数据。常见就这几个：php://filter、php://input、php://stdin
3. 例子：php://filter/convert.base64-encode/resource=flag.php

      	伪协议：php:// 是 PHP 内部专用通道，长得像协议所以叫伪协议。
      				php://filter：专门用来读 PHP 文件源码，不执行代码。
      				Base64 编码：防止浏览器解析乱码，把源码打包成纯文本，解码就能看完整源码。  
      				convert.base64-encode读取完立刻转成 Base64
      				resource=flag.php指定要读的文件是 flag.php
      php://input：读post请求体，执行post请求体里的代码
      data://： 是 PHP 中最灵活的伪协议之一，能直接将代码 / 数据嵌入 URL，无需依赖外部文件，在文件包含漏洞中堪称 “执行代码的万能钥匙”。**核心作用**：将数据直接嵌入 URL，通过**include/require**等函数执行任意 PHP 代码。

tac = 反过来的 cat
• cat 文件名：正序读取文件（正常从上到下读）
• tac 文件名：倒序读取文件（从下到上读）
在 Linux 系统中，* 是一个万能通配符，它的作用是：匹配任意多个字符（包括 0 个字符）。举几个例子你就懂了：
• cat test* → 匹配所有以 test 开头的文件，比如 test.txt、test123.php、test 本身
• cat *.php → 匹配所有 .php 结尾的文件，比如 index.php、flag.php
• cat fla* → 匹配所有以 fla 开头的文件，比如 flag.php、fla123.txt

* 通配符（模糊匹配）
这是 Linux 系统命令（cat/tac/ls） 自带的功能，用来匹配文件名。

**扩展**：URL 编码必须是两位十六进制：
• 正确：%0a
• 错误：%a（服务器不识别）
每一个字符都有对应的十六进制和对应的url编码
✅ 正则过滤的是「原始字符」，根本不过滤「URL 编码里的数字 / 字母」！

*    1.URL 编码字符串：%79（只是一串文本，包含数字 7、9）
  2. 原始字符：hex2bin('79') = 字母 y（正则真正过滤的目标）
  2. 正则会自动解码，但不会进行按位或运算
* hex2bin(十六进制字符串)作用：把 十六进制文本 转换成 对应的原始字符 / 字节
* 用最直白的验证给你钉死这个结论：
  • 十六进制字符串 34 → hex2bin('34') = 字符 4
  • URL 编码 %34 → urldecode('%34') = 字符 4
  👉 两者解码后得到的原始字符一模一样，没有任何区别！
*    1.十六进制 = PHP 代码专用 → 不能放 URL 里
  2. URL 编码 = URL 传输专用 → 专门传输特殊字符
  3. 按位或需要特殊字符（|/~） → 只能用 URL 编码包装传输
* 按位或运算本身返回的是 数字（ASCII 码值），但在 PHP 中，两个「字符 / 字符串」做按位或运算时，最终会自动把这个数字转回成「对应字符」！按位或运算本质是二进制在运算，ASCII码是给人看的
