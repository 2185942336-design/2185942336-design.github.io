## import hashlib

作用：Python 自带的加密库，专门用来算 MD5 / SHA 等哈希值。
• 对应题目：题目里用了 md5($_GET['token'])，我们必须用这个库本地算 MD5。

## md5_hash = hashlib.md5(token.encode()).hexdigest()

### 核心目标

把 `token`字符串转换成标准 32 位 MD5 字符串，与 PHP 的 `md5()`函数结果保持一致。

### 分步拆解

#### 原代码（一行式）

```
md5_hash = hashlib.md5(token.encode()).hexdigest()
```

#### 拆解版（四步走）

```
# 1. 原始字符串
data = token  # 比如 "123", "abc", "000"

# 2. 编码转换（字符串 → 字节）
#    MD5 算法只能处理字节数据，不能直接处理字符串
data_bytes = data.encode()

# 3. MD5 计算
#    hashlib.md5() 是 Python 内置的 MD5 计算器
md5_obj = hashlib.md5(data_bytes)

# 4. 结果格式化（二进制 → 可读字符串）
#    hexdigest() 将二进制结果转为 32 位十六进制小写字符串
md5_hash = md5_obj.hexdigest()
```

### 各组件功能说明

| 组件                | 作用                             | 比喻                         |
| ------------------- | -------------------------------- | ---------------------------- |
| **`token`**         | 待加密的原始字符串               | 要榨汁的苹果                 |
| **`.encode()`**     | 字符串 → 字节转换                | 把苹果切块（MD5 只吃“字节”） |
| **`hashlib.md5()`** | MD5 计算器                       | 榨汁机                       |
| **`.hexdigest()`**  | 二进制结果 → 32 位十六进制字符串 | 把榨出的汁装杯，贴上标签     |

### 比喻记忆

1. 

   **`token`** = 🍎 整颗苹果

2. 

   **`.encode()`** = 🔪 切苹果成块

3. 

   **`hashlib.md5()`** = 🧃 用榨汁机榨汁

4. 

   **`.hexdigest()`** = 🥤 把果汁倒进杯子，贴上“苹果汁”标签

> 你不能直接把整颗苹果扔进榨汁机，得先切块 → 同理，MD5 不能直接吃字符串，得先转成字节。

### 常见误区澄清

- 

  **为什么不用 `hashlib.md5(token)`直接算？**

  → 因为 `hashlib.md5()`只接受字节类型参数，字符串需要先用 `.encode()`转字节。

- 

  **不加 `.hexdigest()`会怎样？**

  → 得到的是一个“MD5 对象”（内存地址），不是我们能读的字符串。

  → 必须用 `.hexdigest()`才能得到 `"202cb962ac59075b964b07152d234b70"`这类可读结果。

- 

  **这个和 PHP 的 `md5()`结果一样吗？**

  → 完全一样，都是标准 32 位小写十六进制 MD5。

### 一句话总结

```
# 这行代码就是：
# “把 token 字符串，算成和 PHP 一模一样的 32 位 MD5 字符串”
md5_hash = hashlib.md5(token.encode()).hexdigest()
```

### 速记口诀

> 一字符串，二转字节，三算 MD5，四转十六进制。

------

**附：完整示例**

```
import hashlib

token = "123"
md5_hash = hashlib.md5(token.encode()).hexdigest()
print(md5_hash)  # 输出: 202cb962ac59075b964b07152d234b70
```

## from itertools import product

翻译：从 Python 自带的工具包 itertools 里，导入一个叫 product 的超级工具。
作用：自动生成 所有字符的排列组合（比如 00、01、ab、1a、fff 等）
product = 全排列组合生成器给它一组字符 + 长度，它就能把所有可能的组合全部给你列出来。专门解决：我要猜所有可能的字符串，但不想手动一个个写 的问题。

标准语法（背会这一句就行）：product(字符集合, repeat=组合长度)
字符集合：你要用来组合的所有字符（比如数字、字母）； repeat：生成几位数的组合（1 位、2 位、3 位...）
例子：1、输入：from itertools import product for i in product('012', repeat=1):    print(i)
                  输出：('0',) ('1',) ('2',)

​			2、输入：for i in product('01', repeat=2):    print(i)
​				  输出：('0','0') ('0','1') ('1','0') ('1','1')

​			3、输入：for i in product('ab', repeat=3):    print(i)
​				  输出：输出所有 8 种组合：aaa/aab/aba/abb/baa/bab/bba/bbb

token_tuple：product 生成的是元组（带括号的），比如 ('1','a')；
''.join(token_tuple)把元组转成正常字符串 1a，才能拿去算 MD5

##  f-string 

* f-string 的强大之处：{} 里可以写函数调用、算术运算等任意代码
* f-string 只有一个铁律：花括号 {} 里可以写任意 Python 表达式，程序会先算出这个表达式的结果，然后把它填回字符串里。
  * 例子：c='A'
  * f'%{ord(c):02x}'     %41      计算 ord('A')=65，格式化为 41，与 % 拼接
  * f'%{ord(c)}'             %65      只计算 ord('A')=65，无格式
  * f'%ord({c}):02x'      %ord(A):02x        把字面文本填进去，无计算

## 编码&解码

* .encode()——文字变字节（编码）——abcdefg—>b'abcdefg'

* .decode()——字节变文字（解码）——b'abcdefg'—>abcdefg

* base64.b64encode(...) —— 把字节变成 Base64 字节

  * Base64 的本质是把二进制数据（任意字节）转成纯文本。
    字符串必须先经过 encode() 变成字节串，才能交给 Base64 处理。

  ```python
  import base64
  
  # ✅ 正确：传入 bytes
  result = base64.b64encode(b"hello")
  print(result)  # b'aGVsbG8='
  
  # ❌ 错误：传入字符串
  result = base64.b64encode("hello")  
  # TypeError: a bytes-like object is required, not 'str'
  ```

  * Base64 的合法字符只有 A-Z a-z 0-9 + / = 这 64 个。

* base64 解码时，每 4 个字符一组解码成 3 个字节。如果总字符数不是 4 的倍数，解码器会自动忽略末尾不足 4 个的零头。这个“丢弃零头”的行为，只发生在解码这种没有标准补 = 机制的脏数据（强行解码一堆混有非法字符的乱码）时。

* 比特（bit）：0或1

* 字节（Byte）：由8个比特组成的固定大小单元

* 3：正常base64编码前，原始字节每3个一组去编码

* 4：正常 Base64 编码后，以及解码器的“胃口”；编码后的字符数永远是 4 的倍数，解码器也 4 个一组解码

* 字节不同个数情况

  * 正常情况：3 个字节 → 4 个字符

    * 字节：Man
    * 二进制：

    ```text
    M: 01001101
    a: 01100001
    n: 01101110
    连成 24 位：010011010110000101101110
    切成 4 个 6 位组：
    010011 → 19 → T
    010110 → 22 → W
    000101 → 5  → F
    101110 → 46 → u
    结果：TWFu（4 个字符，无 =）
    ```

  * 只剩2个字节

    * 字节：Ma（16位）
    * 二进制：

    ```text
    M: 01001101
    a: 01100001
    连起来：0100110101100001
    切成 6 位组：
    010011 (6位) 010110 (6位) 0001 (只有4位)
    最后一组缺了 2 位，用 0 补满 6 位：
    010011 → 19 → T
    010110 → 22 → W
    000100 → 4  → E   ← 末尾补了两个 0
    现在得到了 3 个字符：TWE。
    但解码器需要知道“最后一组有两个位是填充的，不是真实数据”，于是在字符串末尾加一个 =。
    最终输出：TWE=（长度 4，末尾 1 个 =）。
    ```

  * 只剩 1 个字节

    * 字节：M（8 位）
      二进制：

    ```text
    M: 01001101
    切成 6 位组：
    010011 (6位) 01 (只有2位)
    最后一组缺了 4 位，用 0 补满 6 位：
    010011 → 19 → T
    010000 → 16 → Q   ← 末尾补了四个 0
    得到了 2 个字符：TQ。
    因为有两个字节的填充（对应 4 个 0），需要加 两个 = 作为标记。
    最终输出：TQ==（长度 4，末尾 2 个 =）。
    ```


* url编码

  * 原理：把字符转成%+两位十六进制（把字符转换成 % 后面跟这个字符的 ASCII 码的十六进制表示（两位，字母小写）。）
    * URL（网址）里只允许使用一部分字符：英文字母、数字和 -_.~ 等少数符号。
      如果要在 URL 里传中文、空格、/、: 等特殊字符，就必须转义。
  * 目的：让特殊字符能安全在url里传输
  * 可逆性：完全可逆
  * 数据膨胀：把一个字符可能变成3个字符（如    ：（%3A））
  * 常用场景：网址传参、表单提交
  * base64编码：邮件附件、图片内嵌、数据字段传输
  * 一句话区别：URL 编码是为了修路（让字符不撞车），Base64 是为了包装（把任意数据全换成字母数字）。

  * urlencode：

    * urllib.parse.urlencode() 是 Python 自带的一个函数，它不是用来编码单个字符串的，而是用来把字典转换成 URL 查询字符串。

    * ```python
      from urllib.parse import urlencode
      
      params = {'name': '张三', 'age': '18'}
      result = urlencode(params)
      print(result)
      # 输出：name=%E5%BC%A0%E4%B8%89&age=18
      ```

    * 它做的事：
      1. 把字典里的键值对用 = 连起来
      2. 多个键值对之间用 & 连接
      3. 对中文和特殊符号做 URL 编码（字母和数字保持原样）
          它不是你想象中“给随便一个字符串做 URL 编码”的函数。

  * full_url_encode:

    * 是你自己定义的一个函数，做的事完全不同：把传入字符串里的每一个字符，无论字母还是符号，全部变成 %XX 格式。

    * ```python
      def full_url_encode(text: str) -> str:
          return ''.join(f'%{ord(c):02x}' for c in text)
      ```

  * urllib.parse.quote()

    * python标准库自带的用来编码单个字符串的
    * 用法

    ```python
    from urllib.parse import quote
    
    print(quote("php://filter"))
    # 输出：php%3A//filter
    ```

    它会把特殊字符（如 :、/、空格）转成 %XX，但不会动字母、数字、下划线这些安全字符。这是正常的 URL 编码规则。

  ## **双下划线开头结尾的（`__xxx__`）**

  这在Python里叫“魔法方法”或“内置属性”。你可以把它理解为“系统内置的隐藏功能”。普通的变量叫 name，但系统自带的身份证就叫 __name__。你不用记原理，只要记住：看见前后各两个下划线的，就是Python系统预留的特殊钥匙。

  ## 点（.）

  中文叫“的”。A.B 就是“A的B”。"".__class__ 就是“这个空字符串的铭牌”。

  ## 类的“老祖宗”是谁？（object）

  继续用蓝图比喻。在Python的设定里，所有的蓝图（不管别墅图纸、狗窝图纸、汽车图纸），往上追溯，最初都来自一张“终极总图纸”，这张总图纸就叫 object。
  • Python规定，所有类的老祖宗都是 object。因为只有统一了根，才能让所有东西有共同的基础功能。

  ## `object` 不是对象吗？怎么成了类的老祖宗？

  这个问题极其犀利！真相是：在Python里，“类”和“对象”的概念是重叠的。
  • 从“继承”的角度看（血缘关系）：object 确实是所有类的老祖宗（父类）。就像生物界，所有生物都有共同的单细胞祖先。
  • 从“实体”的角度看（实物关系）：object 本身也是一个对象（实例）。因为Python的设计哲学是“万物皆对象”，连类本身也是一个对象（它是 type 这个元类的对象）。
  给你一个绕口令般但绝对正确的结论：
  object 是所有类的父类，但它自己也是 type 类的一个实例对象。
  但在我们这个攻击链里，我们关心的是“血缘继承”，所以把 object 当作老祖宗完全正确。

  ## 什么是“继承链”？（家谱）

  - 比如：**总图纸 `object`** 画了一代车，**二代图纸（汽车类）** 继承了它的轮子，**三代图纸（跑车类）** 又继承了二代的速度。
  - 这个“爷爷→爸爸→儿子”的血缘传递关系，就叫**继承链**。
  - `__mro__` 就是干这个的！它的全称是 Method Resolution Order（方法解析顺序）。`"".__class__.__mro__` 就是让Python打印出这张蓝图的“祖宗十八代”家谱。
  - 因为字符串 `str` 的老爹是 `object`，所以 `__mro__[1]` 就是取家谱里的第二个人——**也就是老祖宗 `object`**。

## 家谱 __mro__ 里，第 1 个（[0]）是谁？为什么跳过去取第 2 个（[1]）？

我们拿 str（字符串类）来打印家谱看看：

```python
print(str.__mro__)  
# 输出结果：(<class 'str'>, <class 'object'>)
```

看清楚这个括号里的顺序了吗？

- **`[0]`（第一个人）**：就是 **`str` 自己**。
- **`[1]`（第二个人）**：才是它的父类 **`object`**。

所以攻击者写 `__mro__[1]`，意思是：**“我不要字符串自己的蓝图，我要往上一级，直接拿那张全宇宙通用的总图纸 `object`！”**
只有拿到了 `object`，才能用接下来的 `__subclasses__()` 去查遍Python里所有加载的类。

## `__subclasses__()`（找后代）

- 我们已经拿到老祖宗 `object` 了。现在要查户口：**“老祖宗啊，你下面总共有多少个子子孙孙（当前内存里加载的所有蓝图）？”**
- `object.__subclasses__()` 就是干这个的，它会返回一个**巨大的列表**，里面装着所有子类蓝图（比如字符串类、列表类、甚至操作系统交互类）。
- `[132]` 就是在这个大列表里数到第133个。在特定的Python版本里，第133个恰好是**能和操作系统对话的类**（叫 `_wrap_close`）。

## .__init__ （房子的装修队）

我们拿到了一张“能与操作系统交互”的蓝图（类）。但蓝图不能直接干活，得找它的初始化方法（即 __init__，你可以理解为“装修函数”）。这个函数负责在盖房时把工具搬进来。
## .__globals__ （装修队的万能工具包）

在Python里，任何一个函数（__init__ 就是个函数）都有一个隐藏属性叫 __globals__。它是什么？
它是一个大字典，里面装着这个函数所在文件（即 os 模块里）所有能用的全局变量和函数。
因为 os._wrap_close 这个类本来就定义在 os 模块里，所以它的 __init__ 函数的 __globals__ 字典里，天然就装着整个 os 模块的所有工具！

## ['popen'] （掏出打电话用的手机）

刚才那个大字典里，有一个工具叫 popen（即 os.popen）。popen 的意思是“打开一个管道给操作系统”。你给它一句Shell命令，它就去执行。
## ("cat /flag") （拨号并下达指令）

调用 popen，把 Linux 下的查看文件命令 cat /flag 传进去。此时，服务器操作系统就乖乖地去读取了这个文件。
## .read() （听筒里传来的回音）

popen 执行完命令后，会返回一个类似“文件流”的东西。.read() 就是把这个流里的所有输出内容（即文件内容）全部读成字符串。

### 总结一下这串“火星文”的完整动作：

1. 找个空字符串，往上查祖宗十八代。
2. 跳过自己，直接找到宇宙总纲 `object`。
3. 让 `object` 把所有后代都叫出来，挑第133个（专门搞系统操作的）。
4. 偷看这个类的装修工具箱，找到连接操作系统的“手机”（`popen`）。
5. 用手机拨号执行 `cat /flag`，并打开免提 `.read()` 把内容念出来。
